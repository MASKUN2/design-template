Notes on shipping small software

The smallest thing that works usually beats the cleverest thing that might. Ship it, watch it, then decide what to add.

Keep the loop short

A fast feedback loop is worth more than any single feature. Optimise for the time between an idea and seeing it run.

build → run → observe → decide

Everything else is negotiable.
