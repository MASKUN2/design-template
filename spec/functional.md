# 4. Functional Requirements

Part of the [{{PROJECT_NAME}} specification](./README.md). Each bullet is one
testable capability, grouped by audience.

## 4.1 Authoring (Author)
- Create a post with a title, a URL slug, and a Markdown body.
- Save a post as a **draft** or **publish** it; change its state at any time.
- Edit any field of an existing post.
- Delete a post.
- See a list of **all** posts — drafts included — with each post's state clearly
  marked.

## 4.2 Reading (Reader)
- Browse a list of **published** posts, newest first.
- Open and read a single published post by its slug.
- Never encounter a draft — not in listings, not by direct link.
